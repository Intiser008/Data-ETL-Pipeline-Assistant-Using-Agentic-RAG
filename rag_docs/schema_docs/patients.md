# Table: patients

Represents each synthetic patient generated by Synthea, containing demographic and identifying details.

---

## Columns

| Column | Type | Description |
| ------- | ---- | ----------- |
| id | UUID | Unique patient identifier (Primary Key). |
| birthdate | DATE | Patient’s date of birth. |
| deathdate | DATE | Patient’s date of death, if applicable. |
| ssn | TEXT | Simulated Social Security Number. |
| drivers | TEXT | Driver’s license number. |
| passport | TEXT | Passport number (synthetic). |
| prefix | TEXT | Name prefix (e.g., Mr., Ms., Dr.). |
| first | TEXT | Patient’s first name. |
| last | TEXT | Patient’s last name. |
| suffix | TEXT | Suffix (e.g., Jr., Sr.). |
| maiden | TEXT | Maiden name. |
| marital | TEXT | Marital status. |
| race | TEXT | Race (white, black, etc.). |
| ethnicity | TEXT | Ethnicity (e.g., Hispanic, Scottish). |
| gender | TEXT | Gender value as stored in the source data. |
| birthplace | TEXT | City or region of birth. |
| address | TEXT | Full postal address string. |

---

## Relationships
- `encounters.patient → patients.id`
- `conditions.patient → patients.id`
- `procedures.patient → patients.id`
- `medications.patient → patients.id`
- `observations.patient → patients.id`
